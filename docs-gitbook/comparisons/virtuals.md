# vs Virtuals Protocol

How Loop compares to and complements Virtuals Protocol.

## Different Goals

| Aspect | Virtuals Protocol | Loop Protocol |
|--------|-------------------|---------------|
| Focus | Agent creation | Human value capture |
| Primary user | Agent creators | Everyday people |
| Value flow | To agent creators | To users |
| Token model | Single ($VIRTUAL) | Dual (Cred + OXO) |

Virtuals asks: "How do we create valuable agents?"

Loop asks: "How do humans capture value from agents?"

## Complementary, Not Competitive

```
┌─────────────────────────────────────────┐
│           Agent Economy                  │
├───────────────────┬─────────────────────┤
│ Virtuals Protocol │   Loop Protocol     │
│                   │                     │
│ Agent creation    │   Value capture     │
│ Agent tokens      │   User vaults       │
│ Creator rewards   │   User rewards      │
└───────────────────┴─────────────────────┘
```

A Virtuals agent could implement Loop to capture value for its users.

## Token Comparison

**Virtuals**: Value accrues to agents and their creators.

**Loop**: Value accrues to users (humans).

Both are valid models for different purposes.

## Integration Possibilities

### Virtuals Agents Using Loop

A Virtuals Service Agent could:
1. Register with Loop AVP
2. Enable capture modules
3. Route captured value to users
4. Earn fees through Loop

### Loop Using Virtuals Agents

Loop users could:
1. Discover Virtuals agents
2. Authorize capture on their behalf
3. Benefit from agent specialization

## When to Use Which

**Virtuals**: Building an AI agent product. Agent is the product.

**Loop**: Capturing value for users. Agent is a servant.

**Both**: Building agents that serve users with creator rewards AND user value capture.
